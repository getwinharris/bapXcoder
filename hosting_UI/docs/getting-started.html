Getting Started with bapXcoder

Complete installation and setup guide for the AI-powered IDE with dual-model architecture

Introduction to bapXcoder

bapXcoder is an advanced, AI-powered Integrated Development Environment featuring the dual-model architecture with persistent project memory. Unlike traditional AI tools that lose context, bapXcoder maintains context across IDE restarts and project switches through its unique internal memory system.

System Requirements

  • Python 3.8+
  • 8GB+ RAM (16GB+ recommended)
  • 5GB+ free disk space
  • Internet connection for model access
  • Git installation

Installation Process

Install bapXcoder using the official installation script:

curl -fsSL https://raw.githubusercontent.com/getwinharris/bapXcoder/main/install.sh | bash

Alternatively, clone the repository manually:

git clone https://github.com/getwinharris/bapXcoder.git
cd bapXcoder
pip install -r requirements.txt
Note: bapXcoder uses a dual-model architecture with internal interpreter and developer functions. The system connects to Hugging Face models via llama.cpp without requiring local downloads.

First-Time Setup

After installation, run bapXcoder for the first time:

./coder.bapx

Configuration

The system will prompt for initial configuration. During setup:

  • Configure your GitHub or Google authentication
  • Select your preferred model access method
  • Set up project memory directory location
  • Customize interface preferences

Project Memory System

bapXcoder creates a .bapXcoder directory in each project for persistent memory. This system maintains:

  • Session continuity across IDE restarts
  • Project-specific context and task tracking
  • File change history and project state
  • AI interaction history and learning

Authentication and Licensing

After the 60-day free trial period, you'll need to subscribe to continue using bapXcoder. The licensing system ensures continuous access to AI features while maintaining your project memory and session state.

Understanding the Dual-Model Architecture

bapXcoder uses two internal specialized models working together:

  • Interpreter Model: Handles communication, UI understanding, multimodal processing, OCR, and context management
  • Developer Model: Specializes in coding tasks, code generation, analysis, and implementation

Frequently Asked Questions

Q: How do I install bapXcoder?

A: Use the curl installation command or clone the repository manually. The installation will set up all necessary dependencies and configuration.

Q: Does bapXcoder work offline?

A: bapXcoder connects to Hugging Face models via llama.cpp. While the models require internet access, your project data stays local for privacy.

Q: How does project memory work in bapXcoder?

A: bapXcoder creates .bapXcoder directories in your projects to store session continuity data, project context, and task history.

Q: What happens after the 60-day trial?

A: After the 60-day trial period, you'll need to purchase a subscription to continue using the full AI features of bapXcoder.

Q: Can I use bapXcoder without internet connection?

A: The core IDE works with local files, but the AI features require internet connection to access models via llama.cpp runtime.
