Overview of CLI Interface

The bapXcoder CLI interface provides direct terminal access to the dual-model AI development environment with full project memory integration. The command-line interface maintains all the features of the web interface while offering the flexibility of terminal-based development workflows.

Core Principle: CLI-first architecture with web UI as an overlay - all functionality available through terminal commands with dual-model AI integration and persistent session memory.

Getting Started with CLI

Basic Commands

Start bapXcoder CLI

./coder.bapx --cli

Starts the bapXcoder IDE in CLI-only mode with dual-model access and session continuity.

Start with Project Directory

./coder.bapx --cli --project /path/to/your/project

Opens bapXcoder IDE in CLI mode targeting a specific project with project memory.

Full CLI with Parameters

./coder.bapx --cli --host 0.0.0.0 --port 8080 --model bapXcoder-VL-8B-Instruct-Q8_0.gguf

Starts CLI with custom host, port, and model settings.

Parameter Type Default Description
--cli flag false Run in command-line only mode without web interface
--project string current directory Specify project directory to work on
--host string 127.0.0.1 Host address for web interface (when not using --cli)
--port integer 7860 Port for web interface (when not using --cli)
--model string bapXcoder-VL-8B-Instruct-Q8_0.gguf Path to GGUF model file
--threads integer 4 Number of CPU threads for model processing
--context-size integer 4096 Context size in tokens for model processing
--gpu-layers integer 0 Number of GPU layers (0 for CPU only)

Frequently Asked Questions

Q: Can CLI mode access both Interpreter and Developer functions?

A: Yes, CLI mode has full access to both internal functions with the same dual-model architecture as the web interface.

Q: Does CLI maintain session continuity like the web interface?

A: Yes, CLI uses the same sessiontree.json and todo.json files for persistent session memory.

Q: Can I switch between CLI and web interface without losing context?

A: Yes, both interfaces share the same project memory system for seamless continuity.

Q: Are CLI commands subject to the same licensing as GUI usage?

A: Yes, all access methods (GUI, CLI, API) consume from the same license quota and subscription tier.

Q: How does file path security work in CLI mode?

A: All CLI operations are constrained to the current project directory with the same security boundaries as GUI.
