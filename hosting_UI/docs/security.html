Security Best Practices

Recommendations for Secure Usage

Project Isolation

  • Always work within dedicated project directories
  • Use the .bapXcoder directory for project-specific memory
  • Never include credentials in files processed by AI
  • Utilize separate projects for different security domains
  • Regularly review file access permissions

AI Interaction Security

  • Always review file operation diffs before approval
  • Verify code changes for security implications
  • Use AI for code review to identify vulnerabilities
  • Request security-specific analysis for authentication code
  • Validate all generated code before deployment

Memory Management Security

  • Regularly audit session memory for sensitive data
  • Clear session memory when working with sensitive projects
  • Use separate user accounts for different security domains
  • Back up project memory separately from code
  • Understand that memory files are stored locally and securely

Key Security Benefit: The dual-model architecture with internal role separation provides inherent safety by ensuring that the Developer function (which handles coding tasks) can only execute instructions that have been validated and approved by the Interpreter function (which handles communication and context management).

Frequently Asked Questions

Q: Does bapXcoder send my source code to external servers?

A: No, your complete project code remains local. Only your text prompts and requests are sent to AI models via encrypted llama.cpp runtime connections.

Q: How are my project memories secured?

A: Project memory is stored encrypted in the .bapXcoder directory with user-specific access controls and session-based encryption.

Q: Can the AI models access files outside my project?

A: No, all file operations are strictly constrained to your project directory with boundary enforcement.

Q: How does the dual-model architecture enhance security?

A: The Interpreter function mediates all operations and validates Developer actions, preventing unauthorized or unsafe operations.

Q: Are my credentials safe when using bapXcoder?

A: Yes, the system has built-in credential detection that automatically prevents exposure of sensitive information to AI models.
