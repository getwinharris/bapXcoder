Troubleshooting Guide

Solutions for dual-model AI IDE with project memory and session continuity

Overview

This troubleshooting guide covers common issues with bapXcoder's dual-model architecture, persistent project memory system, and session continuity features. Solutions are categorized by problem type with specific guidance for the Interpreter and Developer model functions.

Important: Most issues can be resolved by clearing user-specific memory in .bapXcoder/users/{user_id}/ or refreshing your subscription status.

Common Installation Issues

Problem: Installation fails with dependency errors

Possible Causes: Missing system dependencies, incompatible Python version, insufficient permissions

Solution Steps:
1
Verify Python 3.8+ is installed: python3 --version
2
Update pip and install dependencies: pip install --upgrade pip && pip install -r requirements.txt
3
Install llama.cpp dependencies based on your system (CUDA for NVIDIA, Metal for Apple Silicon, etc.)
Problem: Model connection fails during startup

Possible Causes: Network connectivity, Hugging Face rate limits, or authentication issues

Solution Steps:
1
Check internet connection and firewall settings
2
Verify Hugging Face Hub connection: huggingface-cli scan-cache
3
Clear model cache if needed: huggingface-cli delete-cache
4
Restart the application with fresh model connection

Session Continuity Issues

Problem: Session continuity not restoring properly

Symptoms: Continuity message doesn't appear on startup, context lost between sessions

Solution Steps:
1
Check if .bapXcoder/users/{user_id}/sessiontree.json exists and has valid content
2
Verify user authentication and session persistence in todo.json
3
Rebuild session continuity: ai --rebuild-session
4
Restart the IDE to trigger fresh session initialization
Problem: License status not reflecting correctly

Symptoms: Continuity message shows expired license despite active subscription

Solution Steps:
1
Refresh license status: ai --refresh-license
2
Re-authenticate with GitHub/Google OAuth
3
Check Stripe subscription status in admin panel
4
Clear authentication cache and re-login

Model Performance Issues

Interpreter Model (Communication/UI) Issues

Problem: Interpreter model not responding to chat

Possible Causes: Connection issues, context overflow, or model overload

1
Check context length - large conversations can cause slowdowns
2
Try a fresh conversation context to eliminate context overflow
3
Verify multimodal processing capabilities are enabled
4
Check system resources and memory allocation
Problem: Image analysis not working

Symptoms: OCR and visual understanding features failing

1
Ensure image format is supported (PNG, JPG, JPEG, GIF, WebP)
2
Verify vision model components are properly loaded
3
Check that image files are not corrupted or too large
4
Test with sample images provided in the resources directory

Developer Model (Coding) Issues

Problem: Code generation slow or failing

Possible Causes: Complex code analysis, system resources, or context issues

1
Break complex tasks into smaller, specific requests
2
Provide clearer context about existing project architecture
3
Check that the dual-model coordination is working properly
4
Review if the request is being properly routed to Developer function
Problem: Dual-model architecture not separating tasks properly

Symptoms: Coding tasks not being properly routed to Developer function

1
Verify keyword detection for coding tasks ('code', 'function', 'implement', etc.)
2
Check that routing logic is properly detecting coding intent
3
Inspect internal logs for model coordination issues
4
Reset model coordination with: ai --reset-coordination

Project Memory System Issues

Problem: .bapXcoder directory not being created or updated

Possible Causes: Permission issues, existing project memory corruption, or IDE startup problems

Solution Steps:
1
Check directory permissions: ls -la .bapXcoder (if directory exists)
2
Manually create the directory: mkdir -p .bapXcoder/users/{your_user_id}
3
Verify that project memory files are being written: ls -la .bapXcoder/users/{user_id}/
4
Restart IDE with elevated permissions if necessary
Problem: Session memory disappearing after IDE restart

Symptoms: Continuity messages don't reflect previous session

Solution Steps:
1
Check if sessiontree.json and todo.json files are being updated: cat .bapXcoder/users/*/sessiontree.json
2
Verify file write permissions in project directory
3
Look for error messages during IDE shutdown that might prevent memory writes
4
Check that the correct user ID is being used for memory storage

Performance Optimization

Slow Response Times

If responses are taking too long, try these optimizations:

1
Check system resources: htop or Task Manager
2
Reduce context size in settings to improve processing speed
3
Enable GPU acceleration if available (CUDA, Metal, etc.)
4
Use smaller quantization models for faster inference
5
Clear old session data that might be impacting performance

Memory and Resource Management

1
Monitor memory usage with: python -m memory_profiler bapxcoder_local_cli.py
2
Reduce context window size in config.ini for better performance
3
Clear old session files: find .bapXcoder -name "*.json" -newer sessiontree.json -delete
4
Adjust thread counts in settings to optimize for your CPU

Authentication and Licensing Issues

Problem: Authentication fails after 60-day trial period

Possible Causes: Expired trial, subscription processing issues, or license validation problems

Solution Steps:
1
Verify subscription status in admin panel or via Stripe
2
Check that your GitHub/Google account is properly linked
3
Re-authenticate to refresh license information
4
Check that subscription information is properly stored in your GitHub repo
Problem: License countdown not displaying correctly

Symptoms: GUI shows incorrect time remaining or license status

Solution Steps:
1
Refresh license status in the top-right GUI panel
2
Check system clock alignment with internet time
3
Force license refresh via command line: ai --refresh-license
4
Verify license server connectivity if using custom license server

Advanced Troubleshooting

Memory Corruption Recovery

If project memory becomes corrupted, use these steps to recover:

# Backup existing memory cp -r .bapXcoder .bapXcoder_backup # Remove corrupted files (keeping user data structure) rm .bapXcoder/users/*/sessiontree.json rm .bapXcoder/users/*/todo.json # Restart IDE to recreate clean memory files ./coder.bapx

Complete Reset Procedure

If all else fails, perform a complete reset:

1
Stop all bapXcoder processes
2
Backup important session data: cp -r .bapXcoder ~/backup_bapxcoder
3
Remove project memory: rm -rf .bapXcoder
4
Clear authentication tokens in system cache
5
Re-authenticate and allow system to rebuild memory

Frequently Asked Questions

Q: Why does session continuity sometimes fail to restore context?

A: This usually happens due to corrupted session files in .bapXcoder/users/{user_id}/ or authentication issues. Try refreshing your login or rebuilding your session memory.

Q: How do I check if my license is still valid?

A: Check the top-right corner of the GUI interface where license status and remaining days are displayed. You can also use the CLI command: ai --check-license

Q: My project memory isn't persisting after IDE restart. How do I fix this?

A: Verify that .bapXcoder directory exists in your project root and that sessiontree.json and todo.json files exist in your user-specific subdirectory.

Q: Are my project memories stored securely?

A: Yes, all project memories are stored locally in your project directory with user-specific encryption in .bapXcoder/users/{user_id}/

Q: How do I clear project memory if it becomes corrupted?

A: You can delete files in .bapXcoder/users/{user_id}/ leaving only the directory structure, or use the CLI command: ai --reset-memory {user_id}
